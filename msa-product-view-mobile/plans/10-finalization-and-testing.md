# План 10: Финализация и тестирование

## Цель
Завершить разработку приложения, настроить финальные параметры и провести комплексное тестирование.

## Задачи

### 10.1. Настроить Base URL через BuildConfig
- `local.properties`: `API_BASE_URL=https://dev.example.com/api/`
- `secrets.properties` для локальных секретов
- `BuildConfig.API_BASE_URL` для debug и release

### 10.2. Настроить механизм авторизации
- Bearer Token хранится в DataStore
- `AppDataStore` для токена
- `AuthInterceptor` добавляет Authorization header
- `security-crypto` для шифрования токена

### 10.3. Протестировать парсинг JSON из реального ответа API
- Скопировать JSON из DevTools (F12 → Network)
- Проверить маппинг всех полей
- Добавить `@SerialName` при необходимости
- Проверить nullable поля

### 10.4. Проверить работу оффлайн-режима
- Отключить сеть
- Убедиться что кэшированные данные отображаются
- Проверить что приложение не крашится
- Verify Room query работает

### 10.5. Протестировать тултипы на всех типах графиков
- Bar charts: тап на сегмент → тултип
- Pie charts: тап на сектор → тултип
- Donut chart: тап на кольцо → тултип
- Radial chart: тап на бар → тултип

### 10.6. Проверить адаптивность при повороте экрана
- Портретная ориентация
- Альбомная ориентация
- Tablet layout
- Split screen mode

### 10.7. Проверить обработку ошибок
- Offline: показать кэш или сообщение
- HTTP 401: запросить повторный вход
- HTTP 4xx: показать сообщение об ошибке
- HTTP 5xx: показать сообщение о серверной ошибке
- Network Error: показать сообщение о сети
- Parse Error: показать сообщение о формате данных

### 10.8. Убедиться в совместимости с Android 8.0+ (API 26)
- Проверить minSdk = 26
- Тестирование на API 26, 28, 30, 33
- Проверить использование API только с API 26+

### 10.9. Кросс-тестирование на разных размерах экранов
- Small phone (4.7")
- Medium phone (6.1")
- Large phone (6.7")
- Tablet (10")
- Foldable devices
